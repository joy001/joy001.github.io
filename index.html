关于软件测试自动化测试的测试框架

TestNG是Java中的一个测试框架， 类似于JUnit 和NUnit, 功能都差不多， 只是功能更加强大，使用也更方便
Java中已经有一个JUnit的测试框架了。 TestNG比JUnit功能强大的多。 测试人员一般用TestNG来写自动化测试。
开发人员一般用JUnit写单元测试。
官方网站: http://testng.org/doc/index.html

关于hexo的用法

hexo安装

1》安装前提
1.Node.js
2.Git
如果有这两个东西了,就只需要运行命令行 $ npm install -g hexo-cli
如果没有请下载Node.js和Git进行安装,区分window系统和mac系统。
2》建站
安装hexo后,请执行以下命令
$ hexo init <在目录下建的存放的文件名称>
$ cd <在目录下建的存放的文件名称> //cd到该目录下
$ npm install //安装依赖
3》配置文件
具体配置信息参考该链接https://hexo.io/zh-cn/docs/configuration.html
配置最重要的是github的关联。
deploy:
type: git //git类型
repo: https://github.com/joy001/joy001.github.io.git //自己注册的github的账号登录,点击右上角+里面的new repository(名称:github登录名+github.io)
branch: master //可不写
message: ‘frist’ //可不写
4》常用命令
新建一个网站$ hexo init [folder]
新建一篇文章$ hexo new [layout] <标题> //layout 可不写
生成静态文件$ hexo generate
发表草稿$ hexo publish [layout]
启动服务器$ hexo server
部署网站$ hexo deploy
清除缓存文件 (db.json) 和已生成的静态文件 (public)$hexo clean
