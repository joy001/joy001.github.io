关于登录注册

测试登录注意点：1.登录时输入错误密码次数过多时，可能锁定账号或者出现验证码验证登录。
2.其次就是技术方面的防sql注入。
3.然后就是一些等价类，和边界值，如注册时最短长度和最大长度，小于最短长度和大于最大长度。
4.注册时，特殊字符的输入
5.注册时，确认密码与密码不一致
6.注册时，邮箱的格式
7.注册已存在的用户名
8.回车键是否能代替确认
9.tab键是否正确响应进行切换按钮
10.验证系统是否对已经存在用户的用户名大小写转换时是否做出正确的响应

登录：1.用户名密码是否对大小写敏感。大写也能正确登录存在。
2.验证系统是否对在合法用户名或密码前插入空格敏感
3.验证系统是否对在合法用户名或密码后面插入空格敏感
4.验证系统是否允许已经禁止的用户名登录
5.验证系统是否允许已经删除的用户名登录
6.验证系统是否对tab键敏感
7.验证系统是否对用户名或者密码中含有全角字符敏感

不过现在的技术一般不会出现用防sql注入能登录了。

关于hexo的用法

hexo安装

1》安装前提
1.Node.js
2.Git
如果有这两个东西了,就只需要运行命令行 $ npm install -g hexo-cli
如果没有请下载Node.js和Git进行安装,区分window系统和mac系统。
2》建站
安装hexo后,请执行以下命令
$ hexo init <在目录下建的存放的文件名称>
$ cd <在目录下建的存放的文件名称> //cd到该目录下
$ npm install //安装依赖
3》配置文件
具体配置信息参考该链接https://hexo.io/zh-cn/docs/configuration.html
配置最重要的是github的关联。
deploy:
type: git //git类型
repo: https://github.com/joy001/joy001.github.io.git //自己注册的github的账号登录,点击右上角+里面的new repository(名称:github登录名+github.io)
branch: master //可不写
message: ‘frist’ //可不写
4》常用命令
新建一个网站$ hexo init [folder]
新建一篇文章$ hexo new [layout] <标题> //layout 可不写
生成静态文件$ hexo generate
发表草稿$ hexo publish [layout]
启动服务器$ hexo server
部署网站$ hexo deploy
清除缓存文件 (db.json) 和已生成的静态文件 (public)$hexo clean

步骤:
第一步:打开cmd用hexo命令,一路cd到我存放东西的目录下面
第二步:执行hexo server启动服务器,启动服务器后会生成一个本地路径,用本地路径实时查看效果
第三步:打开编写工具类似于webstorm这个工具进行编写博客,用hexo new 标题 命令新建一篇博客。这时候source下面就出现一个刚新建的md文件,就可以写要写的内容了。
第四步:全部编写完的博客,使用hexo generate命令进行生成静态文件
第五步:生成静态文件成功之后就是部署网站,使用hexo deploy命令,成功执行后,本地内容就发布上去了。
